## Backlog — vetted findings without plans (candidates for a next run)

- **BUILD-A (S, MED impact)**: `scripts/generate-embeddings-modal.py:430-438`
  writes `embedding_dimensions` to `version_info` only AFTER all embedding
  inserts; a crash mid-run leaves a partial embeddings table that passes the
  server's shallow `SELECT 1 FROM iconclass_embeddings LIMIT 1` check.
  Currently latent (server's 768 default equals the script's DIMENSIONS),
  but becomes live on any model/dimension change. Fix: write metadata before
  the insert loop + add an embeddings-count sanity check server-side.
- **BUILD-B (S, observability)**: `scripts/build-iconclass-db.py` has
  repeated bare `except Exception: skipped += 1 / pass` blocks (e.g. around
  lines 178-182, 275-281) — a build that silently drops a language section
  still passes server validation. Fix: per-phase skip counters logged at
  exit, warn above a threshold.
- **BUILD-C (M, MED conf)**: chunked release flow has no chunk-count
  manifest; `tryChunkedDownload` (`src/utils/db.ts:82-105`) treats any
  mid-sequence fetch error as end-of-sequence. Gzip integrity + post-download
  validation mitigate in practice (truncated gz fails to gunzip), so impact
  is mostly wasted deploy time, not corruption. Consider a `.manifest` asset
  if chunked assets are kept long-term.
- **TEST-A (S)**: `scripts/tests/test-mcpjam.mjs` (E2E, LLM-driven) is not
  wired into any npm script and silently skips without `ANTHROPIC_API_KEY`.
  Add `test:e2e` script + README note.
- **DX-A (S)**: no `.env.example`; env vars documented only in
  `docs/technical-guide.md`. Low urgency (single-maintainer repo).
- **DX-B (S)**: `test-http-concurrency.mjs` reports failures less verbosely
  than the other two suites (summary count only). Unify failure reporting.
- **CORRECTNESS-A (S, LOW impact)**: in semantic search mode, `totalResults`
  returned to clients is the prefetched candidate count (offset+maxResults
  post-filter), not a true total (`src/api/IconclassDb.ts:654-659`,
  `src/registration.ts:309`). Cosmetic inconsistency vs FTS mode's true total.
- **DOCS-A (decide, don't fix)**: `docs/SKILL.md` version `0.41` vs package
  `0.4.0` — appears to be an intentional independent versioning scheme;

## Findings considered and rejected

- **FTS unbounded candidate set / event-loop blocking**: measured worst-case
  broad terms ("man" → 6,607 notations, "woman" → 4,626) against the real
  DB; synchronous better-sqlite3 handles temp-table insert + join + JS sort
  of that size in milliseconds. Not worth capping; capping would also break
  coverage-first ranking and true `totalResults`. (2026-06-12)
- **`ATTACH DATABASE '${countsPath}'` string interpolation**
  (`src/api/IconclassDb.ts:161`): path is operator-controlled (env var or
  repo default), not request data. By-design; not a vulnerability.
- **Unauthenticated `/debug/memory`**: documented intentional in
  `src/index.ts` (operational signal, matches `/health`).
- **protobufjs/qs reachability**: vulnerable code paths are likely never
  executed in this server (browser ONNX backend; `qs.stringify` not used) —
  which is why Plan 002 is hygiene-priority, not an emergency.
- **Splitting test-tools.mjs into fast/slow suites**: full local suite is
  ~tens of seconds; not enough friction to justify reorganizing 207 tests.
- **Extensions sidecar (feature/extensions-sidecar, issue #373)**: blocked on
  a CC BY-SA-NC licensing decision, not on engineering. A plan can't resolve
  a legal/strategy question; revisit after the maintainer decides.
- **PORT env parsing edge cases** (`parseInt` NaN on malformed PORT): trivial
  operator error surface, fails loudly at listen time.
